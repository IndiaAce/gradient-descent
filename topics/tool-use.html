Tool Use.

The pattern where you give Claude a set of functions it can call mid-response. Claude pauses generation, invokes the tool, gets a result, and continues. This isn't prompt engineering — it's a function-dispatch loop where Claude is the dispatcher and your code is the executor. Active since edition #003.

§ T-01 · CORE THESIS
FUNCTION-DISPATCH LOOP

Claude as the dispatcher.

Tool use is misleadingly named. It sounds like an SDK feature, but mechanically it's a runtime protocol: you describe each tool with a name, description, and JSON schema for inputs. Claude decides when to call it. You implement the logic. The loop runs until stop_reason == "end_turn".

The most immediate use case for Calendra: a Claude agent that reads a verdict JSON and writes a human-readable threat report, optionally calling lookup tools — domain age, URL category — for context the verdict alone can't carry.

The reason this pattern matters more than chat: it's the cleanest way to give a model access to fresh state without retraining or stuffing the context window. Tools are also the API the model uses to do anything in the real world.

// MENTAL MODEL Tool use turns Claude's generation into a state machine. Each state is either "generating text" or "waiting for a tool result." Your code drives the transitions.
// PAIR WITH Workshop for observability — every tool_use block shows up as a fully-typed event in the trace tree. You see exactly which tool fired, with what arguments, returning what.

The loop.

The canonical pattern. Define tools as JSON-schema declarations. Loop until Claude stops. Feed every tool's return value back into the next message turn. Full context from §C-04 ED #005 →

python · the loop // CYCLE 0 
while True:
    resp = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=512,
        tools=tools,
        messages=messages,
    )
    if resp.stop_reason == "end_turn":
        return next(b.text for b in resp.content if hasattr(b, "text"))

    # Claude called a tool — execute and feed back
    results = [execute_tool(b.name, b.input)
               for b in resp.content if b.type == "tool_use"]
    messages += [
        {"role": "assistant", "content": resp.content},
        {"role": "user",      "content": results},
    ]
