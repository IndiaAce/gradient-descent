Pipelines.

The single object that bundles preprocessing and model so leakage becomes impossible by construction. ColumnTransformer routes each column to the right treatment; Pipeline chains it to the estimator; joblib ships the whole thing as one artifact. Introduced this edition as the way Calendra's scattered preprocessing finally becomes one fittable, deployable unit.

§ T-01 · WHY IT MATTERS
CORE THESIS · CALENDRA CONTEXT

Leakage isn't a bug you fix — it's one you design out.

The most common silent failure in an ML notebook: you scale or impute on the whole dataset, then split. Now the test fold has seen the training set's mean and variance. Cross-validation scores come back beautiful — and lie. The model looks better than it will ever be in production.

A Pipeline makes this structurally impossible. It chains every preprocessing step to the estimator into one object. When CV re-fits it on each fold, the scaler learns from that fold's training rows only. You stop relying on remembering to split first. The architecture remembers for you.

ColumnTransformer is the routing layer: numeric columns to StandardScaler, categorical to OneHotEncoder(handle_unknown="ignore"), text to a vectorizer — each branch fit independently, outputs concatenated.

The deployment payoff is just as large:

  • pipe.fit(X, y) — preprocessing + model, one call
  • joblib.dump(pipe, ...) — scaler travels with model
  • pipe.predict(raw_df) — no manual scaling, ever
  • gbm__max_depth — tune nested params by name
  • handle_unknown="ignore" — unseen TLD won't crash
// THE BEHAVIORAL UNLOCK Once preprocessing lives inside the estimator, the gap between "works in the notebook" and "works at the FastAPI endpoint" disappears. One artifact does both.

§ T-03 · CODE ARCHIVE
3 PATTERNS · GROWS WITH EDITIONS

Reach-for-it patterns.

// PATTERN 01 · LEAKPROOF WRAP

Numeric scaled, categorical one-hot, glued to the model. CV now re-fits preprocessing per fold. Source: §C-02 ED #006.

python · pattern 01 · leakproof // THE CORE IDEA 
pre = ColumnTransformer([
    ("num", StandardScaler(), numeric),
    ("cat", OneHotEncoder(handle_unknown="ignore"), nominal),
])
pipe = Pipeline([
    ("pre", pre),
    ("gbm", GradientBoostingClassifier(random_state=42)),
])
pipe.fit(X_tr, y_tr)   # scaler learns from X_tr only

// PATTERN 02 · HONEST CV + PERSIST

Cross-validate the whole pipeline, then dump it as one artifact. The reloaded object scales for you. Source: §C-03 ED #006.

python · pattern 02 · cv + joblib // ONE ARTIFACT 
cv = cross_val_score(pipe, X, y, cv=5, scoring="roc_auc")
print(cv.mean())                # preprocessing fit inside each fold

pipe.fit(X_tr, y_tr)
joblib.dump(pipe, "models/calendra_pipeline.joblib")

reloaded = joblib.load("models/calendra_pipeline.joblib")
reloaded.predict(raw_df)        # no manual scaling, ever

// PATTERN 03 · TUNE NESTED PARAMS

The step__param grammar reaches inside the pipeline, so Optuna or GridSearch can tune the model without unwrapping it. Source: §C-05 ED #006.

python · pattern 03 · step__param // REACHES INSIDE 
pipe.set_params(
    gbm__n_estimators=300,      # step "gbm", param "n_estimators"
    gbm__max_depth=4,
    pre__num__with_mean=False,    # nest deeper: step__sub__param
)
# same grammar GridSearchCV + Optuna trials use
