§ T-01 · WHY IT MATTERS
CORE THESIS · CALENDRA CONTEXT

The loop that turns runs into diffs.

You have a working GBM hitting 98% accuracy. But which n_estimators got you there? What max_depth did you finally settle on? When you retrain on fresh labeled data next month, how will you know the new run is actually better — or just differently broken?

Without experiment tracking, model development is archaeology. Every Saturday begins with reconstructing what past-you did from notebook cells and vague memory. The number you remember was the best of one path through the search space — and you have no record of the paths you didn't take.

MLflow fixes this. Every training run becomes a named, permanent record: parameters, evaluation metrics, saved model artifact, source git commit, environment. The UI lets you sort, filter, and visually diff any two runs side by side.

The API surface you actually need is tiny:

  • set_experiment(name) — bucket runs under a project
  • start_run(run_name=...) — open one record
  • log_param / log_metric — fill it in
  • log_model — pickle the artifact
  • mlflow ui — sort, diff, deploy
// THE BEHAVIORAL UNLOCK Once tracking is free, you start running five short experiments per session instead of one long manually-tweaked one. That's when you actually learn what moves the needle.

§ T-03 · CODE ARCHIVE
3 PATTERNS · GROWS WITH EDITIONS

Reach-for-it patterns.

// PATTERN 01 · BASELINE LOG

The minimum viable wrap. Take your existing training cell and add five lines. Source: §C-02 ED #005.

python · pattern 01 · baseline // 5 LINES 
mlflow.set_experiment("calendra-gbm")

with mlflow.start_run(run_name="baseline"):
    model.fit(X_train, y_train)
    mlflow.log_params(params)
    mlflow.log_metric("auc_roc", auc)
    mlflow.log_metric("f1", f1)
    mlflow.sklearn.log_model(model, "model")

// PATTERN 02 · CONFIG SWEEP

A list of configs, each becomes a named run. Diff them in the UI by sorting on metric columns. Source: §C-03 ED #005.

python · pattern 02 · sweep // N RUNS, ONE SCRIPT 
experiments = [
    {"run_name": "shallow-fast",  "max_depth": 3, "learning_rate": 0.10},
    {"run_name": "deeper-slower", "max_depth": 5, "learning_rate": 0.05},
]

for cfg in experiments:
    run_name = cfg.pop("run_name")
    with mlflow.start_run(run_name=run_name):
        model = GradientBoostingClassifier(**cfg, random_state=42)
        model.fit(X_train, y_train)
        mlflow.log_params(cfg)
        mlflow.log_metric("f1", f1_score(y_test, model.predict(X_test)))
        mlflow.sklearn.log_model(model, "model")

// PATTERN 03 · FEATURE AB TEST

Log two runs with different feature sets — bare baseline and embedding-augmented. Then sort the comparison view by AUC and accept reality.

python · pattern 03 · feature A/B // FOR ED #005 ICEBREAKER 
for name, feats in [("baseline", X_base), ("with-embed", X_with_embed)]:
    with mlflow.start_run(run_name=name):
        model = GradientBoostingClassifier(**best_params, random_state=42)
        model.fit(feats, y_train)
        mlflow.log_metric("auc_roc", roc_auc_score(y_test, model.predict_proba(feats_test)[:, 1]))
        mlflow.log_param("n_features", feats.shape[1])
        mlflow.sklearn.log_model(model, "model")
