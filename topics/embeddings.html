§ T-01 · CORE THESIS
SEMANTIC FEATURE AXIS

The language around a URL is the signal.

Your Calendra feature pipeline uses url_count as a raw integer. Useful, but brittle: a single obfuscated link scores the same as a zero-URL benign invite, and a verbose phishing description with no URLs at all gets ignored entirely.

The description field itself contains far richer signal. Attackers write things like "Click here to confirm your attendance: http://malcal-invite.xyz/rsvp?t=abc123" — the surrounding language is semantically suspicious before you ever parse the URL.

Sentence transformers map raw text into a 384-dim vector encoding semantic meaning. The cosine similarity to a centroid of known-malicious description embeddings becomes a single new feature — or use the raw embedding dims via PCA.

// L2 EVIDENCE arxiv 2602.22433: 14 sentence-transformer models tested on attack-to-CVE mapping. multi-qa-mpnet-base-dot-v1 hit F1 = 89.0, precision 84.0, recall 94.7 on MITRE ATT&CK. The finding: description text outperforms TTP summary labels — surrounding language carries more weight than structured tags.
// MODEL CHOICE For Calendra's ~5k sample corpus, all-MiniLM-L6-v2 is the sweet spot — 384 dims, fast on CPU. Bigger encoders overfit cosine-similarity centroids in < 1k examples.

§ T-03 · CODE ARCHIVE
CENTROID SUSPICION FEATURE

The recipe.

Encode every description. Average the known-malicious vectors to get a centroid. Each description's cosine similarity to that centroid is a single new feature — works as a drop-in alongside url_count.

Source: §C-05 ED #005. See the Calendra GBM project page for how this slots into the feature matrix.

python · description_embed.py // CENTROID 
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity

model = SentenceTransformer("all-MiniLM-L6-v2")  # 384d, fast

embeddings = model.encode(descriptions)   # (n, 384)

# cosine similarity to centroid of known-malicious embeddings
mal_centroid   = embeddings[labels == 1].mean(axis=0, keepdims=True)
suspicion_feat = cosine_similarity(embeddings, mal_centroid).flatten()

# drop straight into your feature matrix alongside url_count, etc.
X_aug = np.hstack([X_base, suspicion_feat.reshape(-1, 1)])
